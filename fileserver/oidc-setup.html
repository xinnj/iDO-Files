My Files

Authentication Required

This service requires authentication (AUTH_REQUIRED=true), but the OpenID Connect (OIDC) authentication provider is not set up yet. Follow the steps below to configure it.

1

Configure OIDC Environment Variables

Set the following environment variables in your deployment configuration. Variables marked required must be provided; others have sensible defaults.

OIDC_CLIENT_ID — Your OIDC client identifier (required)
OIDC_CLIENT_SECRET — Your OIDC client secret (required)
OIDC_DISCOVERY_URL — Your OIDC provider's discovery endpoint (required)
OIDC_REDIRECT_URI — Callback path after login (default: /fileserver/login)
OIDC_LOGOUT_PATH — Logout endpoint path (default: /fileserver/logout)
OIDC_LOGOUT_REDIRECT_URI — Redirect after logout (default: /download)
OIDC_SSL_VERIFY — Verify SSL for OIDC requests (default: yes)
OIDC_REDIRECT_URI and OIDC_LOGOUT_REDIRECT_URI accept relative paths (e.g. /fileserver/login) or absolute URLs (e.g. https://files.example.com/fileserver/login). Use absolute URLs when running behind a reverse proxy to ensure the IdP redirects to the correct public address.
2

Using Helm Chart

If deployed with Helm, edit the OIDC ConfigMap:

kubectl edit cm file-server-oidc

Fill in the required values and adjust the defaults if needed:

OIDC_CLIENT_ID: "your-client-id"
OIDC_CLIENT_SECRET: "your-client-secret"
OIDC_DISCOVERY_URL: "https://your-oidc-provider/.well-known/openid-configuration"
OIDC_REDIRECT_URI: "/fileserver/login"
OIDC_LOGOUT_PATH: "/fileserver/logout"
OIDC_LOGOUT_REDIRECT_URI: "/download"
OIDC_SSL_VERIFY: "yes"
3

Using Docker

Pass environment variables when starting the container:

docker run -e AUTH_REQUIRED=true \
  -e OIDC_CLIENT_ID=your-client-id \
  -e OIDC_CLIENT_SECRET=your-client-secret \
  -e OIDC_DISCOVERY_URL=https://your-oidc-provider/.well-known/openid-configuration \
  -e OIDC_REDIRECT_URI=/fileserver/login \
  -e OIDC_LOGOUT_PATH=/fileserver/logout \
  -e OIDC_LOGOUT_REDIRECT_URI=/download \
  -e OIDC_SSL_VERIFY=yes \
  ...
4

Restart the Service

After configuration, restart the file server pod or container for the changes to take effect.

kubectl rollout restart deployment
To make this bucket publicly accessible without authentication, set the AUTH_REQUIRED environment variable to false.
